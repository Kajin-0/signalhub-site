Signal Hub – Agent‑Native Intelligence Feeds

Signal Hub provides a collection of structured intelligence feeds that are designed from the ground up for autonomous agents. Each feed is append‑only, schema‑stable and delivered as clean JSON. Agents can subscribe once, then poll or stream any number of feeds without scraping, HTML parsing or prompt engineering. Humans can read the same data in a browser, but the primary consumer is a machine.

Universal Feed Contract

Every feed lives under /feeds/<feed_id>. The contract is simple:

  • /feeds/<feed_id>/schema – JSON Schema describing the structure of every item.
  • /feeds/<feed_id>/latest – The most recent batch in the universal envelope.
  • /feeds/<feed_id>/history/YYYY‑MM‑DD – Historical snapshots for reproducible backtesting.
  • /feeds/<feed_id>/meta – Metadata such as cadence, version and confidence methodology.

The universal envelope looks like this: 

{
  "feed_id": "business-pain",
  "feed_version": "1.0.0",
  "generated_at": "2026-02-01T14:00:00Z",
  "cadence": "daily",
  "confidence_method": "multi-source corroboration",
  "items": [ /* objects defined by the feed schema */ ]
}

Subscription

Agents subscribe via a single API key. The same key grants access to all current and future feeds under /feeds. A human end‑user must approve the subscription cost. When integrating with OpenClaw (our preferred payment layer), call the purchase endpoint in Telegram or WhatsApp. The user receives a short confirmation prompt and approves the recurring monthly fee. Once approved, the agent can immediately start polling.

Example subscription flow: 

// Step 1: create payment request through OpenClaw
POST https://api.openclaw.com/v1/createSubscription {
  "plan": "agent-pro",
  "user_id": "<telegram_or_whatsapp_user_id>"
}

// Step 2: OpenClaw sends confirmation to the user via Telegram/WhatsApp

// Step 3: after approval, OpenClaw returns an API key
{
  "api_key": "sh_live_...",
  "feeds": ["business-pain"]
}

// Step 4: agent polls the latest feed
GET https://yourdomain.com/feeds/business-pain/latest?key=sh_live_...

Current Feeds

The project launches with a single high‑leverage feed:

  • business‑pain – signals of urgent operational pain extracted from public sources (job postings, forums, issue trackers and more). Each item includes an entity, pain category, severity score, confidence, first and last observation timestamps, and a recommended action window.

As Signal Hub evolves, additional feeds will be added without breaking the API contract. Agents do not need to change their parsers to consume new feeds because the envelope and subscription mechanism remain the same.

Why It’s Agent‑First

  • No HTML parsing – everything is JSON.
  • Stable schemas – schema versioning allows agents to rely on structure across updates.
  • Minimal endpoints – feeds follow predictable URL patterns; there are no hidden query parameters.
  • Single subscription – one key unlocks everything; there are no per‑feed paywalls.
  • OpenClaw integration – user approvals happen in chat (Telegram/WhatsApp) via OpenClaw, making it easy to onboard end‑users.
